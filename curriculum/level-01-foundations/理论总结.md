# 理论基础总结：LeCun 世界模型路线

> 读完这篇就等于读了 LeCun 2022 + World Models 2018 的核心思想。15 分钟。

---

## 1. 核心问题：为什么 Generative AI 走错了路？

LeCun 的核心论点：**逐像素预测是浪费**。

想象你走进一个房间，你看到左边有一把椅子，右边有一张桌子。你的大脑需要"生成"椅子和桌子的每一个像素才能理解这个场景吗？不需要。你的大脑提取了抽象表示："椅子在左，桌子在右"。

但 GPT / DALL·E / Sora 做的事情恰恰相反——它们试图预测/生成每一个像素、每一个 token。这是巨大的计算浪费，而且大多数像素和 token 对人类理解世界毫无意义。

**类比**：
- 预测股票价格 → 有用（语义层面）
- 预测股票价格的"第几位小数是奇数还是偶数" → 没用（像素层面）

JEPA 的核心思想：**不预测像素，预测表示（representations）**。在一个抽象的、压缩的潜在空间里做预测。

## 2. 架构：六个模块

LeCun 设想的自主智能体由六个模块组成：

```
                    ┌──────────────┐
                    │ Configurator │  ← 注意力路由器：决定其他模块做什么
                    │  (配置器)     │     这是整个系统的"CEO"
                    └──────┬───────┘
           ┌───────────────┼───────────────┐
           ▼               ▼               ▼
    ┌────────────┐  ┌────────────┐  ┌────────────┐
    │ Perception │  │   World    │  │    Cost    │
    │  (感知)     │  │   Model    │  │   (成本)    │
    │            │  │  (JEPA)    │  │            │
    │ 编码器把    │  │ 预测世界    │  │ 评估：当前   │
    │ 感官输入    │  │ 会如何变化   │  │ 状态有多"好" │
    │ 变成表示    │  │            │  │ 或多"危险"   │
    └────────────┘  └─────┬──────┘  └─────┬──────┘
                          │               │
                          ▼               ▼
                    ┌──────────────────────────┐
                    │         Actor             │
                    │        (行动器)            │
                    │  根据 World Model 的预测    │
                    │  和 Cost 的评估来选择动作    │
                    └──────────────────────────┘
                               │
                               ▼
                    ┌──────────────────────────┐
                    │   Short-Term Memory       │
                    │      (短期记忆)            │
                    │  记住"刚刚发生了什么"       │
                    └──────────────────────────┘
```

**六个模块的分工**：

| 模块 | 做什么 | 类比 |
|------|--------|------|
| **Perception** | 感官→表示。看/听到的东西变成向量。 | 眼睛和耳朵 |
| **World Model** | 预测。给定当前状态+动作→预测下一状态。 | 想象力和常识 |
| **Cost** | 评估。当前状态有多好？离目标多远？有危险吗？ | 情绪和直觉 |
| **Actor** | 决策。找到最小化 Cost 的动作序列。 | 运动皮层 |
| **Configurator** | 注意力路由。设置其他模块的子目标。 | 前额叶（执行控制） |
| **Short-Term Memory** | 记住最近的状态序列。 | 工作记忆 |

**关键点**：Configurator 是最神秘、实现最少的模块。它相当于"你现在应该关注什么"——这个注意力机制还没有好的实现。

## 3. JEPA：核心引擎

JEPA（Joint Embedding Predictive Architecture）= World Model 的具体实现。

### 3.1 基本流程

```
输入 x (一张图，遮住一部分)
     │
     ▼
┌─────────────┐        ┌─────────────┐
│ Context      │        │ Target       │
│ Encoder      │        │ Encoder      │
│ (处理可见部分) │        │ (处理被遮部分)  │
│              │        │ STOP-GRADIENT│  ← 关键！梯度不传回 Target
└──────┬──────┘        └──────┬──────┘
       │ ctx_latent            │ tgt_latent
       │ (上下文表示)            │ (目标表示：这是"正确答案")
       ▼                       │
┌─────────────┐                │
│ Predictor    │                │
│ (从上下文     │                │
│  预测目标)    │                │
└──────┬──────┘                │
       │ pred_latent            │
       │ (预测的表示)             │
       ▼                       ▼
       └─────────┬─────────────┘
                 │
            L2 Loss (pred_latent - tgt_latent)
```

**用大白话说**：
1. 给你看半张图（Context），让你猜另外半张图"长什么样"（Target）
2. 但你不是猜像素长什么样，而是猜一个**压缩表示**（latent）
3. 正确答案由 Target Encoder 提供（但它的参数不通过梯度更新，用 EMA 慢慢跟）
4. 你通过 Predictor 从上下文生成预测

### 3.2 为什么 Target Encoder 要 STOP-GRADIENT？

这是 JEPA 最巧妙的设计。如果 Target Encoder 也通过梯度更新，系统会**坍缩**（collapse）：两个编码器合谋输出全零向量 → Loss = 0 → 但什么都没学到。就像考试的时候，考生和出题人是同一个人——他可以"预测"出100分的答案（直接把答案抄过来就行）。

**解法**：
- Target Encoder 不通过反向传播更新
- 它通过 EMA（指数移动平均）慢慢跟随 Context Encoder 的参数
- 这保证了 Target 始终提供"稳定的正确答案"，Context 和 Predictor 必须努力去追赶

### 3.3 和 Autoencoder 的区别

| | Autoencoder (MAE) | JEPA |
|---|---|---|
| 预测什么 | 像素值 | 潜在表示（latent） |
| 损失函数 | 像素级 MSE | 潜在空间 L2 |
| 是否"浪费" | 是（预测无关像素） | 否（只预测语义） |
| 坍缩问题 | 无（像素不会坍缩） | 有（需要 Stop-Grad + EMA）|

## 4. 能量模型（EBM）：为什么需要它？

JEPA 解决了一个问题（不预测像素），但引入了一个新问题：**怎么知道预测好不好？**

在像素空间，你可以直接比较像素值。在潜在空间，你需要一个"好坏评判标准"——这就是**能量函数**。

```
E(x, y) = 低  →  x 和 y 是一对好的匹配（如：上下文和目标的正确配对）
E(x, y) = 高  →  x 和 y 不匹配（如：上下文不应该是这个目标）
```

JEPA 的 Predictor 本质上在**最小化能量**——它试图找到让能量最低的预测。

目前 JEPA 用简单的 L2 距离当能量函数，但 LeCun 认为需要更复杂的能量模型来：
1. 处理多模态（同一上下文对应多个可能的未来）
2. 提供不确定性估计
3. 在潜在空间做规划（用能量评估不同动作序列的好坏）

## 5. 层级规划：为什么需要多层抽象？

LeCun 的架构有多个层级（H-JEPA = Hierarchical JEPA）：

```
高层：  "去厨房拿杯子"      ← 抽象子目标（秒-分钟级）
  │
中层：  "走向桌子，伸右手"    ← 运动原语（毫秒-秒级）
  │
低层：  "激活二头肌，旋转手腕"  ← 肌肉控制（毫秒级）
```

每一层有自己的 JEPA 世界模型：
- **高层世界模型**：预测"拿了杯子"之后厨房的状态变化
- **低层世界模型**：预测手腕旋转 15 度后的下一帧画面

不同层级用不同的时间尺度和抽象程度。这就是 Configurator 的职责——它决定"现在应该在哪一层思考"。

## 6. 到底和 Dreamer/Sora 有什么区别？

### JEPA vs Dreamer

| | Dreamer | JEPA |
|---|---|---|
| 预测什么 | 下一帧状态（含像素重建损失） | 下一帧的**潜在表示** |
| 训练目标 | 重建 + 奖励预测 + KL 散度 | 纯潜在空间预测 |
| 计算效率 | 中等（重建浪费算力） | 高 |
| 规划 | ✅ 在想象中规划（MPC） | ❌ 目前还不直接支持规划 |

**趋势**：Dreamer-CDP（2026）去掉了重建损失，正在向 JEPA 靠拢。

### JEPA vs Sora

| | Sora | JEPA |
|---|---|---|
| 什么类型 | 生成式（扩散模型） | 预测式（JEPA） |
| 输出 | 完整视频帧（像素） | 潜在表示（向量） |
| 是否世界模型？ | LeCun：不是（只是模式匹配） | LeCun：是（但还不完整） |
| 为什么不是世界模型？ | Sora 不会"推理"——它只是记住了训练数据中的模式。把杯子推下桌子 → Sora 不会生成杯子碎掉的画面（除非训练数据里有），因为它不懂物理。 |

## 7. 从 World Models (2018) 到 JEPA (2023)：进化路径

**Ha & Schmidhuber (2018) 的贡献**：
- 第一次证明"在世界模型里做梦"可行——Agent 可以在学到的世界模型里训练策略，不需要和真实环境交互
- 架构：VAE（压缩）→ MDN-RNN（预测）→ Controller（决策）
- 缺点：VAE 重建像素浪费算力；RNN 无法处理长期依赖

**LeCun (2022) 的改进**：
- 保留"在世界模型里规划"的核心思想
- 用 JEPA 替换 VAE：不再重建像素，只在潜在空间预测
- 用 Transformer 替换 RNN：更好的长期依赖和并行训练
- 增加层级结构（Configurator + 多层世界模型）

**一句话总结**：LeCun 拿了 Ha & Schmidhuber 的"世界模型规划"框架，去掉了像素重建，换上了潜在空间预测，再加上层级注意力路由。

## 8. 当前研究状态（2026）

```
✅ 已实现：                 🟡 部分实现：              🔴 未实现：
   I-JEPA (图像预测)           LeWorldModel (端到端)      Configurator (注意力路由)
   V-JEPA 2 (视频预测)         Hierarchical Planning      完整的 Cost Module
   MC-JEPA (运动分解)          DiLA (动作分解)            多层 H-JEPA
   A-JEPA (音频)               Belief-space JEPA          开放世界泛化
   Point-JEPA (3D)             SkyJEPA (实物机器人)
   CNN-JEPA (轻量化)
```

**最大缺口**：Configurator（注意力路由器）基本没有实现。这是连接"被动预测"和"主动决策"的缺失环节。
